cutwave

Beat-cut your b-roll,
reframe your videos —
without leaving your machine.

Two modes. Long-form: drop in a song and some b-roll clips, and cutwave finds the beat, cuts the footage to it, crops out any on-screen text it spots, and can burn in a cinematic caption that pulses with the music. YouTube Short: drop in one existing video, and it auto-picks the most engaging window, tracks the subject (face first, motion as fallback), and reframes it to fill a vertical screen. Both run entirely on a local Python + ffmpeg pipeline — nothing you drop in gets uploaded anywhere.

What it actually does

No cloud calls, no ML models to download — a from-scratch numpy/scipy/OpenCV pipeline plus the system ffmpeg.
〜

Beat detection, from scratch

Band-pass filters the song to the kick/bass range, builds an onset-strength envelope, and peak-picks beat times. Tempo comes from autocorrelating that envelope — no beat-tracking library involved.

✂

Cuts that respect the beat

The beat grid becomes a list of shot boundaries with a minimum and maximum shot length, so the edit feels intentional instead of chaotic.

▭

Text-avoiding crop

OpenCV's MSER detector finds glyph-shaped clusters (captions, watermarks) without OCR. A 2D search over crop position and zoom finds the window of your chosen aspect ratio that overlaps them the least.

◐

Shot quality scoring

Frame-differencing and brightness/contrast checks favor footage with real motion and reasonable exposure, and steer away from frozen or blown-out frames.

Aa

Cinematic beat-synced captions

Captions are composited frame-by-frame in Python: a PIL text sprite scales and shears from an exponential "hit" envelope timed to each beat, with a lagging drop-shadow layer for cheap parallax depth.

◈

Live 3D caption preview

In the browser, a Web Audio analyser reads the finished video's own audio and drives real CSS 3D transforms on a caption overlay — a bonus layer on top of the already-rendered file.

YouTube Short mode

Feed it one existing video instead — no song, no b-roll.
◎

Auto highlight selection

Videos longer than ~75s get scanned in full with the same motion/brightness/contrast scoring as the b-roll picker, and the most engaging ~45s window is used. Shorter videos are reframed whole.

◍

Face tracking, motion fallback

OpenCV's built-in YuNet face detector finds the subject when there's a face on screen; when there isn't (gameplay, action, general b-roll), the largest moving region takes over instead.

▥

Screen-filling reframe

The tracked position is EMA-smoothed and interpolated per frame into a 9:16 crop that follows the subject, with a slow continuous push-in for some cinematic motion even on static shots.

Run it locally

This is a local tool by design — the editing pipeline needs ffmpeg and a real Python process, which a static site can't run. Clone it and start it on your own machine.
Install ffmpeg

macOS: brew install ffmpeg. It must be on your PATH.

Clone the repo

git clone https://github.com/ShimonDeitel/cutwave.git && cd cutwave

Run it

./run.sh — creates a virtualenv, installs the pinned dependencies, and starts the app.

Open it

http://localhost:3000 — pick Long-form or Short at the top, drop your files, and go.

git clone https://github.com/ShimonDeitel/cutwave.git
cd cutwave
./run.sh
# → http://localhost:3000
Why this page isn't the app.

GitHub Pages only serves static files — it can't run the Python + ffmpeg backend that does the actual beat detection, text-avoiding crops, and video rendering. This page is a static landing page describing the project; the real app runs on your machine via ./run.sh above.
